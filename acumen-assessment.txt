Docker Desktop (running)

Git 
docker-compose --version 

WHAT YOU'LL BUILD A data pipeline with 3 Docker services:
Flask API - Mock customer data server 
FastAPI - Data ingestion pipeline 
PostgreSQL - Data storage 

Flow: Flask (JSON) → FastAPI (Ingest) → PostgreSQL → API Response

PART 1: FLASK MOCK SERVER Build Flask REST API serving customer data from JSON file on port 5000.
Data File: data/customers.json 
Create 20+ customers with: 
customer_id, first_name, last_name, email, phone address, date_of_birth, account_balance, created_at 

Endpoints 
Endpoint Method Description 
/api/customers GET Paginated list (page,limit params) 
/api/customers/{id} GET Single customer 
/api/health GET Health check 

Response Format { "data": [...], "total": 20, "page": 1, "limit": 10 } 
Requirements Load from JSON file (not hardcoded) 
Pagination support 404 for missing customers

Dockerfile + requirements.txt

PART 2: FASTAPI PIPELINE Build FastAPI service that ingests Flask data into PostgreSQL using dlt library on port 8000. 
Database Table: 
customers 
customer_id VARCHAR(50) PRIMARY KEY 
first_name VARCHAR(100) NOT NULL 
last_name VARCHAR(100) NOT NULL 
email VARCHAR(255) NOT NULL 
phone VARCHAR(20) 
address TEXT 
date_of_birth DATE 
account_balance DECIMAL(15,2) 
created_at TIMESTAMP 

Endpoints
1. POST /api/ingest Fetch all data from Flask (handle pagination) Upsert into PostgreSQL 
Return: {"status": "success", "records_processed": 20}
2. GET /api/customers Query params: page, limit Return paginated results from database 
3. GET /api/customers/{id} Return single customer or 404 

Requirements 
SQLAlchemy model 
Auto-pagination from Flask 
Upsert logic (update if exists) 
Error handling 
Dockerfile + requirements.txt
